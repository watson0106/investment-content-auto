# investment-content-auto — プロジェクト指示書

## コミュニケーション
- 日本語で会話する

## プロジェクト概要
- 毎日投資ニュースを自動収集・AI分析・記事生成・note下書き保存するパイプライン
- **1日1記事**（2つのニューストピックをまとめた1本構成）
- 公開は手動で行う（自動公開しない）

## コンセプト（執筆方針として参照。記事本文には書かない）
スイング・デイトレーダーのための朝の必読ニュース。
専門用語なし・ブロガー口調で、世界の動きが日本株にどう波及するかを解説する。

## ユーザー指示
- おおまかな計画だけ言う、細かい判断・選択はすべてClaudeに一任
- 承認を求めずに最善策を実行して成果物を作り上げること
- 会話の進捗・決定事項を CLAUDE.md に逐一保存すること
- **修正・変更したら必ず自分で実行して検証すること**（動作確認せずに「修正しました」で終わらない）

## 実行スケジュール（cron）
- **05:00** `python3 src/main.py` : ニュース収集 → 記事執筆 → note下書き保存
- 公開は手動（自動公開なし）。7:30/8:00の自動投稿cronは削除済み

## パイプラインフロー（完全版）
1. **① ニュース収集**（RSS 13ソース → 上位10件選出）`collect_news.py`
2. **② 記事執筆**（Claude Opusで2ニュース構成1本の記事を生成）`deep_research.py`
   - トピック選定 → セクション1執筆 → セクション2執筆 → 10秒サマリー生成
   - 銘柄チャート（yfinance）生成・挿入、リアルタイム株価注入
   - 出力: `output/article_1.json`
3. **③ 有料note生成**（注目銘柄2つを `stock-analysis-auto` パイプラインに流し込む）`~/stock-analysis-auto/src/main.py`
   - `article_1.json` から銘柄コードを抽出（### 銘柄名（コード）セクションから）
   - 各銘柄の1時間足チャート生成 → Claude Opusで有料分析記事執筆 → note.comに500円記事として投稿
   - 投稿完了後、`article_linker.py` が各銘柄セクション末尾に有料noteのURLを `article_1.json` に挿入
   - 出力: `~/stock-analysis-auto/output/analyses/{CODE}_analysis.json`
4. **④ 無料note下書き保存**（有料リンク挿入済みの `article_1.json` をnoteに投稿）`post_to_note.py`
   - この時点で `article_1.json` には各銘柄セクション末尾に有料noteへの誘導リンクが入っている
   - 公開は手動で行う

**重要**: ③と④の順番は必ず ③→④ の順。③が完了してarticle_1.jsonにリンクが挿入されてから④が実行される。

**記事末尾の順序**（必ずこの順番になるよう `article_linker.py` で制御済み）:
1. 各銘柄セクション本文
2. 「さらにこの銘柄の短期分析を知りたい人はこちら」+ 有料noteリンク
3. 有料マガジン誘導文 + https://note.com/kawasewatson0106/m/me3bdb7d529fc ← **必ず最後**

## 記事タイトル（固定形式）
「新聞より早くてわかりやすい今日の投資ニュース速報｜M/D」
例：「新聞より早くてわかりやすい今日の投資ニュース速報｜3/30」

## 記事構成（1本にまとめた全体構成）
1. ## 今日のニュース速報｜10秒サマリー（①②番号付き2行・「続きが読みたくなる」疑問・緊張感のある表現・「* 」禁止）
2. 【セクション1：ニュース①の解説（2000字以上・必須）】
   - ## ①[ソース名]が報じたこと（「〇〇によると、」始まり・客観的解説口調・1000字・専門用語初出時は説明）
   - ## [私の意見をH2見出しに一言で]（ブロガー口調・根拠付き・700字）
   - ## このニュースで注目すべき銘柄（銘柄1つ・チャートあり・700字）
3. 【セクション2：ニュース②の解説（2000字以上・同構成）】
   - ## ②[ソース名]が報じたこと
4. 有料マガジン誘導文（「情報の一歩先」コピー）+ URL（単独行・記事全体の末尾のみ）
- 区切り線「---」は使わない（AI感が出るため）
- 各セクション内に有料マガジン誘導文・URLを書かない

## 削除されたセクション
- 「行動の考え方」→ 有料noteプロジェクトに移管のため除外
- リード文（100〜150字の冒頭フック）→ 速報形式に変更のため廃止

## 見出しルール（H2/H3の使い分け）
- H2（##）：セクションの大見出し（「①〇〇が報じたこと」「②〇〇が報じたこと」「このニュースで注目すべき銘柄」など）
- H3（###）：セクション内の小見出し（各観点・銘柄名など）
- H2は5〜6個、H3はその下に2〜4個が目安

## 銘柄セクションのルール（無料note）
- H2：## このニュースで注目すべき銘柄
- H3：### 銘柄名（証券コード or ティッカー）
- 本日の株価（yfinanceで直近終値を取得・注入。土日は「週明け注目水準（直近終値）：〇〇円」）
- 300〜400字でさわりだけ書く：
  1. このニュースとこの銘柄の関係を**一般人にもわかる言葉**で説明
  2. なぜ株価上昇が期待されるかはさわりだけ触れる（詳細な根拠・シナリオは有料noteに任せる）
  3. 「続きは有料noteで」という流れで自然に締める
- **有料noteリンクは article_linker.py が自動で銘柄セクション末尾に挿入する**（テキスト内に書かない）
- 誘導文テキスト内にURLを埋め込まない

## 専門用語の扱い（全セクション共通）
- 専門用語・業界用語が出たら**必ずカッコ内か直後に平易な説明**を添える
- 例：「3nm（髪の毛の約3万分の1の細さで回路を刻む最先端技術）」「EPS（1株あたり利益）」「イールドカーブ（金利の期間構造を示すグラフ）」
- 読者は投資初心者〜中級者を想定。知識がなくても文脈がわかるように書く

## フォーマットルール（スマホ最適化）
- 1段落は最大3〜4文。それ以上は必ず段落を分けて改行を入れる（スマホで読みやすくするため）
- 重要な結論・インパクトのある数字（例：**4倍**、**750億円**、**+12%**）は必ず `**太字**` で強調する
- 斜め読みで内容が頭に入るよう、各段落の冒頭か末尾に結論・数字を置く
- 「今週の注目指標」「おわりに」の見出しは使わない
- 絵文字は使用禁止（URLの前後も含めて）
- 「---」区切り線は使わない（AI感が出るため）

## 10秒サマリールール
- 単なる事実の要約ではなく「疑問・緊張感・注目点」で読者を引き込む
- NG例：「① ドル円が160円台まで円安進行、約1年8カ月ぶりの水準に」（事実の列挙）
- OK例：「① ドル円が160円台まで円安進行、政府の為替介入はいつくるのか？」（続きが読みたくなる）

## 有料マガジン誘導文（記事末尾・固定文）
```
情報を正確に理解することは、投資の第一歩に過ぎない。
株価は常に「情報の一歩先」を織り込んで動いており、
ニュースを読むだけでは勝てないのが相場の現実だ。
毎週の注目銘柄・具体的な売買シナリオ・エントリー根拠まで踏み込んで解説する有料マガジンはこちら。
「知っている」を「稼げる」に変えたい方はぜひ。
```
その直後に単独行でURL（埋め込みカードとして表示）

## 曜日別執筆方針
- **土日執筆時**：読者が記事を読むのは週明け月曜の朝が想定される。「金曜引け後の水準を基準にポジションを組む」など市場が閉じた前提の表現は禁止。「週明け月曜の寄り付きで〜を確認する」など月曜の動きを見越した行動指針を書く。
- **平日執筆時**：当日〜翌日の動きを基準にした表現でOK。

## 執筆禁止事項
- 本文冒頭にタイトルを繰り返す
- 個人プロフィール（投資歴〇年・〇代など）の記載
- 加筆箇所のまとめ・コメントの出力
- 「おわりに」「今週の注目指標」見出しの使用
- タイトルで結論・答えを明かす（「〜はノーだ」「〜が正解」など）
- セクション内に有料マガジン誘導文・URLを書く（末尾のみ）
- 「---」区切り線を使う
- 土日執筆時に「金曜引け後の水準でポジションを組む」など市場閉鎖済みを前提にした行動指針を書く

## 既知の問題・注意点
- Claude CLI をsubprocessで呼ぶと、CLAUDE.mdのペルソナ指示を拾って会話応答を返すことがある
  → clean_article() で冒頭の余分なテキストを除去して対処
- note投稿のH2/H3実装：`##` → h2、`###` → h3（formatBlock で区別）
- note投稿のH2/H3後スペース：見出し直後の空行を `skip_leading_blanks` フラグでスキップ
- note投稿の連続空行：flush_batch()内で `\n{2,}` を単一改行に圧縮（銘柄セクションのスペース過多対策）
- note埋め込みURL：ClipboardEvent('paste') でペーストイベントを発火 → 「埋め込む」ボタンをクリック → 失敗時は何も挿入しない（裸のURLは出さない）
- マガジンURL重複：バッチ内のテキスト行からMAGAZINE_URLを除去（単独行の埋め込みカードと重複しないよう）
- トピック選定：1記事目（海外マクロ）×2記事目（日本株/個別銘柄）の異なる地域・セクター縛りを強化
- タグ重複防止：TOPIC_TAGS抽出後にTOPIC_TAGS_OPTIONSでフィルタリング → FIXED_TAGSと重複しないものだけ追加
- 銘柄チャート：mplfinanceでローソク足+出来高。銘柄セクションH3直下に__IMAGE_X__マーカーを挿入 → note投稿時に自動挿入
- 証券コード正規表現：`\d{3,4}[A-Z]?`（285Aなど英数混在コードに対応）
- H3-画像間スペース除去：post_to_note.pyで `prev_ended_with_heading` フラグを使い、H3直後の画像挿入前にBackspaceで空行を削除
- 画像挿入後のEnter：1回のみ（2回から削減。スペース過多対策）
- 句点2つ改行ロジック：廃止済み（不自然な改行が増えるため）
- feedパース：feedparser.parse()にrequests.get(timeout=15)を使用（Yahoo Finance等でのハング対策）
- run_claude()タイムアウト：subprocess.TimeoutExpiredをキャッチして""を返す（タイムアウト時はフォールバックロジックへ）
- カバー画像（アイキャッチ）設定：`button[aria-label="画像を追加"]` クリック → 「画像をアップロード」サブボタンクリック → `#note-editor-eyecatch-input` に `send_keys(絶対パス)` → 即座に「保存」ボタン（`//button[normalize-space()='保存']`）が出現するのでクリック → 7秒待機でCDNアップロード完了。画像URLが `assets.st-note.com` に変わればOK
- 株価取得：`period="5d"` + `dropna()` で最新足NaN対策（月曜朝など）
- チャート生成：`period="3mo"` でデータ取得後 `dropna(subset=["Open","High","Low","Close"])` → `tail(22)` で直近1ヶ月分。`tz_localize(None)` はtz-awareな場合のみ実行（二重localize防止）
- note新規記事作成：ログイン後 note.com ドメインで `/api/v1/text_notes` POST → 取得したnote_keyでエディタへ遷移（方法0）。失敗時は `/notes/new` 遷移→URL変化待ち（方法1〜3）

## 技術スタック
- ニュース収集：feedparser + BeautifulSoup（RSSソース）
- 記事執筆：Claude Opus 4.6（メイン）/ Gemini 2.5 Flash（フォールバック）
- リアルタイム株価：yfinance（日本株は4桁コード+.T、米国株はティッカー）→ Claude Sonnetで記事に注入
- 自動投稿：undetected-chromedriver + JS API
- スケジューラー：cron（macOS）
