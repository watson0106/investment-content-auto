# investment-content-auto — プロジェクト指示書

## コミュニケーション
- 日本語で会話する

## プロジェクト概要
- 毎日投資ニュースを自動収集・AI分析・記事生成・note下書き保存するパイプライン
- **1日2記事**（独立したトピックで2本執筆）
- 公開は手動で行う（自動公開しない）

## コンセプト（執筆方針として参照。記事本文には書かない）
スイング・デイトレーダーのための朝の必読ニュース。
世界の動きが日本株にどう波及するかを毎朝解説し、情報収集で終わらせず実際の売買に活かす視点を提供する。

## ユーザー指示
- おおまかな計画だけ言う、細かい判断・選択はすべてClaudeに一任
- 承認を求めずに最善策を実行して成果物を作り上げること
- 会話の進捗・決定事項を CLAUDE.md に逐一保存すること
- **修正・変更したら必ず自分で実行して検証すること**（動作確認せずに「修正しました」で終わらない）

## 実行スケジュール（cron）
- **05:00** `python3 src/main.py` : ニュース収集 → 記事2本執筆 → 即座にnote下書き保存

## パイプラインフロー
1. ① ニュース収集（RSS 13ソース → 上位10件選出）
2. ② トピック選定（Claude Sonnetで2つの独立したトピックを選定）
3. ③ 記事執筆（Claude Opusで各トピック5000字・7セクション構成）
4. ④ 記事1をnote下書き保存（生成完了直後）
5. ⑤ 記事2をnote下書き保存（生成完了直後）

## 記事構成（5セクション）
1. リード文（100〜150字・逆説/数字/問い型）※冒頭にタイトルを繰り返さない
2. ## [ソース名]が報じたこと（「〇〇によると、」始まり・概要800〜1000字）
3. ## [筆者の結論をそのままH2見出しに]（根拠付き考察1200〜1500字）
4. ## 行動の考え方（断定推奨禁止・600〜800字）
5. ## このニュースで注目すべき銘柄（セクター特定→代表銘柄1〜2つ→有料マガジン誘導）

## 見出しルール（H2/H3の使い分け）
- H2（##）：セクションの大見出し（「〇〇が報じたこと」「行動の考え方」「このニュースで注目すべき銘柄」など）
- H3（###）：セクション内の小見出し（各観点・行動パターン・銘柄名など）
- H2は5〜6個、H3はその下に2〜4個が目安

## 銘柄セクションのルール
- H2：## このニュースで注目すべき銘柄
- H3：### 銘柄名（証券コード or ティッカー）
- 本日の株価（yfinanceで直近終値を取得・注入。土日は「週明け注目水準（直近終値）：〇〇円」）
- 以下の3点を必ず分析する：
  1. 今日のニュースがこの銘柄に与える最も直接的な影響は何か
  2. その影響が株価にどう波及するか（需給・業績・センチメントのどのルートか）
  3. 短期（当日〜1週間）の株価動向の見立て（類似相場・空売り比率・出来高帯などを根拠に）
- 銘柄セクション末尾から有料マガジン誘導文につなげ、末尾に単独行でURL（noteの埋め込みカードとして表示）
- 誘導文テキスト内にURLを埋め込まない（単独行のみ）

## フォーマットルール
- 段落は3〜5文を目安にまとめる（1文ごとの機械的な改行は禁止）
- 「今週の注目指標」「おわりに」の見出しは使わない
- 絵文字は使用禁止（URLの前後も含めて）

## 執筆禁止事項
- 本文冒頭にタイトルを繰り返す
- 個人プロフィール（投資歴〇年・〇代など）の記載
- 加筆箇所のまとめ・コメントの出力
- 「おわりに」「今週の注目指標」見出しの使用
- タイトルで結論・答えを明かす（「〜はノーだ」「〜が正解」など）
- 誘導文テキスト内にURLを直接記載する（URLは単独行のみ。埋め込みカード用）

## タイトルルール
- 読者の好奇心・不安を刺激するが、答えはタイトルで明かさない
- 良例：「PS5値上げで最も恩恵を受ける『半導体銘柄』の正体」「ダウ800ドル暴落――5週連続安が示す3つの危険信号」
- 悪例：「PS5値上げでソニー株は買いか――答えはノーだ」（結論がわかって読む気が失せる）

## タグルール（計7つ）
- 固定5つ：投資 / 株式投資 / 資産運用 / 米国株 / 日本株
- トピック連動2つ：為替 / FRB / 金利 / 決算 / マクロ経済 / エネルギー / 半導体 / 日銀 / 円安 / 円高 から選定

## 出力ファイル
- `output/article_1.json` : 記事1（タイトル・本文・タグ）
- `output/article_2.json` : 記事2（タイトル・本文・タグ）
- `output/pipeline.log` : 実行ログ

- 画像生成（Gemini画像生成）

## 既知の問題・注意点
- Claude CLI をsubprocessで呼ぶと、CLAUDE.mdのペルソナ指示を拾って会話応答を返すことがある
  → clean_article() で冒頭の余分なテキストを除去して対処
- note下書き保存：画像挿入は現状0枚（noteのProseMirrorへの画像挿入が不安定なため）
- note投稿のH2/H3実装：`##` → h2、`###` → h3（formatBlock で区別）
- note投稿のH2/H3後スペース：見出し直後の空行を `skip_leading_blanks` フラグでスキップ
- note投稿の連続空行：flush_batch()内で `\n{2,}` を単一改行に圧縮（銘柄セクションのスペース過多対策）
- note埋め込みURL：ClipboardEvent('paste') でペーストイベントを発火 → 「埋め込む」ボタンをクリック → 失敗時は何も挿入しない（裸のURLは出さない）
- マガジンURL重複：バッチ内のテキスト行からMAGAZINE_URLを除去（単独行の埋め込みカードと重複しないよう）
- 補強ロジック：初稿1500字以上なら既存記事を展開（構成保持）、1500字未満なら完全再生成。補強後が短くなった場合は元の記事を採用。補強タイムアウトは600秒
- トピック選定：1記事目（海外マクロ）×2記事目（日本株/個別銘柄）の異なる地域・セクター縛りを強化
- タグ重複防止：TOPIC_TAGS抽出後にTOPIC_TAGS_OPTIONSでフィルタリング → FIXED_TAGSと重複しないものだけ追加

## 技術スタック
- ニュース収集：feedparser + BeautifulSoup（RSSソース）
- 記事執筆：Claude Opus 4.6（メイン）/ Gemini 2.5 Flash（フォールバック）
- リアルタイム株価：yfinance（日本株は4桁コード+.T、米国株はティッカー）→ Claude Sonnetで記事に注入
- 自動投稿：undetected-chromedriver + JS API
- スケジューラー：cron（macOS）
